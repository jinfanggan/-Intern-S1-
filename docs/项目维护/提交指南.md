## 一、官方接口契约（提交的硬要求）

正式评测**不用**我们的 runner/main，只做三件事：

```python
from user_agent import ReasoningAgent      # 提交根目录必须有 user_agent.py
agent = ReasoningAgent(client=official_client)   # client 由平台提供，禁止硬编码 key
result = agent.solve(problem, metadata)     # → dict，必须含非空字符串 final_response
```

- `official_client.chat(messages, temperature, max_tokens) -> str`（同步）
- 返回 `{"final_response": str, "trace": [...]}`；`final_response` 是**干净的最终答案**（非调试日志），`trace` 可 JSON 序列化
- 本项目已适配：`submission/user_agent.py` 是契约薄壳，内部复用 `jiushao/` 引擎（见 `jiushao/adapter.py`）

## 二、提交包结构

平台从提交根目录加载，需自包含：

```
<提交根>/
├── user_agent.py        # ← 来自本仓库 submission/user_agent.py
├── jiushao/             # ← 本仓库引擎包（整个目录）
├── requirements.txt     # 官方依赖 + 我们的（sympy / math-verify / scipy）
└── （可选）tools/ 等辅助文件，均用相对路径
```

**不要**放进提交包：`.env`、`runs/`、`datasets/`、`reference/`、`__pycache__/`、虚拟环境。

## 三、组装提交包（从本仓库到官方 baseline fork）

```bash
# 1. fork 官方 baseline 到自己账号，clone 下来
git clone git@github.com:<你的账号>/Challenge-Cup-2026.git
cd Challenge-Cup-2026

# 2. 用本项目的入口 + 引擎替换 baseline
cp <本仓库>/submission/user_agent.py ./user_agent.py
cp -r <本仓库>/jiushao ./jiushao

# 3. 合并依赖：官方 requirements.txt 追加我们的
cat >> requirements.txt <<'EOF'
sympy>=1.12
math-verify
scipy
EOF

# 4. 干净环境自测（务必！评测环境是干净的）
python3.10 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
export INTERN_API_KEY="你的书生token"
python main.py --input_file sample_data/dev.jsonl --output_dir sample_outputs
#   检查 sample_outputs/*.json 的 final_response 是否为干净答案
```

## 四、提交方式

官方支持两种（以最新通知为准）：
1. **GitHub 授权提交**（判分系统 7 月上线后）：推送到 fork 仓库 → 在判分系统授权 → 提交**仓库地址 + commit SHA**（不是分支名）
2. **邮件提交**（判分系统上线前）：打包 `.zip` 发送至官方邮箱

**关键约束**：
- 提交频率：**每天最多 2 次**（12:00 定时出分）、**每周最多 10 次** → 必须本地充分验证再提交
- 正式评测用**闭源测试集**，不返回逐题反馈 → 依赖我们的本地代理评测集调试

## 五、提交前自检清单（官方 + 本项目）

**接口**
- [ ] 提交根目录有 `user_agent.py`，可被 `python -c "from user_agent import ReasoningAgent"` 导入
- [ ] `ReasoningAgent.__init__` 至少接受 `client` 参数
- [ ] `solve(problem, metadata)` 返回 dict，含**非空字符串** `final_response`
- [ ] `final_response` 是干净答案（非日志/堆栈）；`trace` 可 JSON 序列化
- [ ] 无硬编码 API key（client 由平台注入）

**依赖与环境**
- [ ] 干净虚拟环境安装 `requirements.txt` 无报错
- [ ] 无仅本机存在的隐藏依赖；关键版本已固定
- [ ] 提交包无 `.env`/`runs`/缓存/大文件

**运行**
- [ ] `python main.py --input_file sample_data/dev.jsonl --output_dir sample_outputs` 跑通
- [ ] 抽查输出 `final_response` 是合理答案

**本地一键自检命令**
```bash
python -c "from user_agent import ReasoningAgent; print('导入 OK')"
python -m pytest -q     # 若把 tests/ 也带进去
```

## 六、模型与流控（提交相关）

- 主力模型：`intern-s2-preview`（数学最强）。平台 client 侧统一管理，我们代码不写死
- 本地调试流控：默认 RPM 10/TPM 5000 不够，备注"挑战杯"申请 **RPM 100 / TPM 80万**（依据见 [OFFICIAL_FINDINGS.md](../../OFFICIAL_FINDINGS.md)）
- S2 单次约 53s / 5000 token → 本地大样本调试建议用 gpt-5-nano，S2 仅做关键验证

## 七、常见判分异常（官方列举）

| 异常 | 原因 |
|---|---|
| 仓库授权失败 | 提交了分支名而非 commit SHA |
| 类缺失 | `user_agent.py` 或 `ReasoningAgent` 不在根目录 / 不可导入 |
| final_response 无效 | 缺失、为空、或无法 JSON 序列化 |
| 运行失败 | 依赖冲突、超时、硬编码 API key |
