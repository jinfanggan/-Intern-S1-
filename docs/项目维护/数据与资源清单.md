## 二、数据集（`../datasets/`，开发期代理评测集）

正式评测集是隐藏的（官方不发），以下是我们自备的公开 benchmark，覆盖相同的研究生级 18 子领域风格，**全部带标准答案**，用于开发期量化迭代。

| 数据集 | 路径 | 条数 | 答案类型 | 覆盖领域 | 在代码中的 loader |
|---|---|---|---|---|---|
| 官方样例 | `reference/Challenge-Cup-2026/sample_data/dev.jsonl` | 3 | 整数/分数 | 抽象代数/测度积分/复分析 | `dsets.load("official")` |
| TheoremQA | `datasets/TheoremQA/data/test-*.parquet` | 747（纯文本）| float/int/bool/list | 350+ 定理，含复分析/拓扑 | `dsets.load("theoremqa")` |
| HARDMath | `datasets/HARDMath/data/HARDMath.json` | 1060 | 渐近表达式 | 非线性 ODE/积分/WKB/边界层 | `dsets.load("hardmath")` |
| ARB(math) | `datasets/ARB/data/arb_math_*.json` | 121 | 数值/符号 | 研究生级，拓扑/复分析/ODE | `dsets.load("arb")` |
| MATH-500 | `datasets/MATH-500/test.jsonl` | 500 | 各类 | 高中竞赛（仅冒烟用） | `dsets.load("math500")` |

**各数据集字段格式**（loader 已统一为 `Problem(id, question, answer, subject, meta)`）：
- TheoremQA parquet 列：`Question / Answer / Answer_type / Picture`（带图题已在 loader 跳过）
- HARDMath JSON 键：`question / answer_val / question_type / small_eval_point / small_analytical / large_eval_point / large_analytical`（后四个是数值判分点，喂给数值裁判）
- ARB JSON 键：`Problem_Statement / Final Answer / Topic / Output Format Instructions`
- 官方样例 JSONL：`{idx, problem, answer, subject, source}`（`answer` 仅本地调试，正式评测不传）

### 如何重新获取数据集

详见 `../datasets/README.md`（含每个数据集的 HuggingFace/GitHub 地址与 SOTA 成绩表）。要点：
```bash
# HF 数据集（TheoremQA / MATH-500 等）
export HF_ENDPOINT=https://hf-mirror.com   # 国内镜像（HF 直连慢时用）
python -c "from huggingface_hub import snapshot_download; snapshot_download('TIGER-Lab/TheoremQA', repo_type='dataset', local_dir='datasets/TheoremQA')"
# gated 数据集（Putnam-AXIOM / HLE）需 HF 账号授权后带 token 下载
```

| 项 | 获取方式 | 用途 |

- `requirements.txt`：lagent 钉死 commit `0ab2e2f5` + ray/openai/anthropic

## 四、运行产物与缓存（`runs/`，不入库，可重建）

```
runs/
├── cache/<model>/<hash>.json   # LLM 调用磁盘缓存（断点续跑 + 消融复用的关键）
└── <run_name>/
    ├── meta.json               # 运行配置（数据集/模型/profile/时间）
    ├── results.jsonl           # 每题摘要（id/pred/gold/correct/elapsed），断点续跑依据
    └── events/<题id>.jsonl     # 每题完整事件流（debug + 前端回放 + 赛规留痕）
```

- **缓存键**：`sha256(model + messages + kwargs + cache_tag)`，与并发无关；删 `runs/cache/` 即清缓存重跑
- **断点续跑**：`runner` 启动时读 `results.jsonl` 跳过已完成题
- **离线重判**：判分器升级后 `python scripts/rejudge.py <run_name>` 零 API 成本重判
- **事件流回放**：`python scripts/view_log.py <run_name> <题id>`
