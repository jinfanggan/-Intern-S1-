## 一、定位

挑战杯·书生赛道 **XH-202627「基于 Intern-S 系列模型的数学智能体设计与推理创新」** 参赛系统。

- **赛题本质**：底模固定为 Intern-S 系列 API（不训练、不微调），分差只来自智能体工程。
- **核心任务**：实现 `agent.solve(problem, metadata) -> dict`，对研究生级 18 子领域数学题给出正确答案（占分 60%）。
- **代号「九韶」**：取秦九韶《数书九章》（算法化巅峰）+「箫韶九成」（多声部协奏 = 多智能体协作）。

## 二、技术栈

| 层 | 技术 |
|---|---|
| 语言 | Python 3.10+（async/await） |
| 模型访问 | OpenAI 兼容 API（开发期 gpt-5-nano；目标 intern-s2-preview） |
| 数学工具 | SymPy / SciPy / NumPy / math-verify（沙箱内） |
| 求解范式 | SC-TIR（多链工具集成推理）+ 数值裁判过滤 |
| 测试 | pytest + pytest-asyncio（FakeLLM 离线，不打网络） |

## 二.1 数据流（单题生命周期）

```
题目 → ① 路由(规划)  → ② N 链 TIR 求解（推理+真实代码执行）
                          ↓ N 个候选
       ③ 数值裁判过滤（构造可执行裁判 → 剔除被数值否决的候选）
                          ↓ 幸存候选（裁判不适用则全保留）
       ④ 等价归一投票 → 选最大簇 → final_response + trace
```
事件全程写 `runs/<run>/events/<id>.jsonl`（debug + 前端回放 + 赛规留痕）。

## 四、官方接口契约（提交必须满足）

```python
# submission/user_agent.py
class ReasoningAgent:
    def __init__(self, client, *args, **kwargs): ...      # client 平台给，不硬编码 key
    def solve(self, problem: str, metadata: dict) -> dict # 返回须含非空 final_response，推荐 trace
```
- 官方 client 接口：`client.chat(messages, temperature, max_tokens) -> str`（同步）
- 适配层 `adapter.OfficialClientLLM` 把它包成引擎所需的 async 接口（内存缓存，不写磁盘）
- baseline 仓库：`github.com/InternLM/Challenge-Cup-2026`（fork 后替换 user_agent.py + jiushao/）

## 五、关键流程（求解 → 模块映射）

```
solve(problem) [adapter.solve_problem]
  → solve_tir × N        [solver]      # 多链并行，每链 写码→沙箱→回填
  → construct_oracle      [oracle]      # 构造数值裁判（数值求解原问题）
  → oracle_filter         [adapter]     # 剔除被裁判否决的候选（带适用性自检）
  → cluster_and_vote      [aggregate]   # 等价归一投票
  → final_response + trace
```

## 六、配置与运行

```bash
cp .env.example .env          # 填 OPENAI_API_KEY（开发）/ INTERN_API_KEY（比赛）
pip install -r requirements.txt
python -m pytest -q           # 111+ 用例应全绿
python scripts/run_eval.py --dataset official --profile b2     # 官方样例
python scripts/compare_oracle.py official 3                     # 裁判对比
```

消融 profile：`b0`(裸CoT) / `b1`(+TIR) / `b2`(+8链投票) / `b3-oracle`(裁判闭环)。
